# Corex usage rules

Corex is a Phoenix UI library: function components and HEEx for markup, JavaScript hooks under `assets/` for Zag.js state machines.

- Prefer `Corex.Content.new/1` (and related list helpers) for list-driven components.
- Register hooks in `assets/js/app.js` and match `phx-hook` names to the hook modules.
- API and patterns: [hexdocs.pm/corex](https://hexdocs.pm/corex).

For deeper agent workflows, use the `corex` skill generated by `mix usage_rules.sync` when `package_skills: [:corex]` is configured.
